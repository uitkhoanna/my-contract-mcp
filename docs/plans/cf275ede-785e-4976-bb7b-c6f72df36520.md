## Context
`solidity-auditor-mcp` is an MCP stdio server (Node ≥18, CommonJS, one runtime dep `@modelcontextprotocol/sdk`) that audits Solidity contracts via the Cysic Minimax model `minimax-m3` at `https://token-ai.cysic.xyz/v1/chat/completions` (OpenAI-compatible, Bearer auth). Four tools: `audit_contract`, `check_vulnerability`, `gas_optimization`, `generate_report`. The pipeline is a 3-pass flow (recon → deep scan → severity scoring) with SWC mapping (curated registry in `src/swc.js`, 37 entries, plus category-alias table) and large-contract chunking (`LARGE_SOURCE_CHARS = 80_000`, `MAX_CHUNKS = 6`, padded overlap at top-level declaration boundaries).

Files in scope: `server.js`, `src/cysicClient.js`, `src/auditor.js`, `src/prompts.js`, `src/swc.js`, `examples/Vulnerable.sol`, `examples/demo.md`, `README.md`, `ARCHITECTURE.md`, `package.json`, `.env.example`, `docs/`.

Repository review reveals the code is largely mature: `cysicClient.js` already has a per-request `AbortController` timeout (`DEFAULT_TIMEOUT_MS = 60_000`), retries on `5xx` and `429` with backoff (250ms / 750ms via `backoffMs(i)`), and a loose JSON parser that strips ```json fences, balances outer braces, and repairs smart quotes and trailing commas. The API key is only read from `process.env.CYSIC_API_KEY` and is never hardcoded; the `Authorization: Bearer ${apiKey}` header is only constructed in `_postWithTimeout`. All four tools have strict string validation in `server.js` (`stringField`) and `auditor.js` (per-tool guards, 500k char cap). `package.json` `files` whitelists `server.js`, `src/`, `examples/`, `README.md`, `ARCHITECTURE.md`, `.env.example`. `.gitignore` excludes `.env` and `.humanize/`.
